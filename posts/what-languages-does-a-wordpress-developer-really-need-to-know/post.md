Working with WordPress goes far beyond installing themes and plugins. If the idea is to act as a developer (and not just an advanced user), there are minimum technical requirements that you need to master — and that doesn't mean knowing everything, but knowing enough to understand, modify and create safely and autonomously. 

🔹 HTML + CSS 
Base of the entire structure and style. Knowing how to structure content, apply styles and understand good practices is the first step. 

🔹PHP 
WordPress is built on PHP. Even if you don't create plugins from scratch, you'll need to understand how themes and functions work behind the scenes. 

🔹 JavaScript (basic to intermediate) 
Many modern themes use JS for interactivity. And with the block editor (Gutenberg), knowledge of JS has become even more relevant. 

🔹 MySQL (basic) 
Knowing how the WordPress database is structured helps you create custom queries, filters and optimizations. 

🔹 WordPress Core, Hooks and Template Hierarchy 
Knowing what a filter, action is, how functions.php works, the template hierarchy... is what separates a customizer from a developer. 

🔹 Git + good versioning practices 
If you are going to collaborate on projects or maintain a professional portfolio, knowing Git is essential. 

🔹 Minimum knowledge of security and performance 
It doesn't need to be DevOps, but it needs to avoid mistakes: SQL Injection, sanitization, validation, correct use of scopes and native resources.
